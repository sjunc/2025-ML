복습 

지난 주 배운 방식으로 데이터셋 임의 작성후 선형모델로 예측
K-NN 방식으로 교체 후 예측 ( 더 나은 결과 도출)

이상치가 있을 때 더 나은 방식이 무엇인지 선택하는 것과 
사전에 데이터에 대한 필터링이 중요 

머신러닝 프로젝트 과정 요약
1. 데이터 분석
2. 모델 선택
3. 훈련 데이터로 모델을 훈련
4. 새 데이터에 모델을 적용해 예측 - 추론
 (일반화가 잘 된 모델)

데이터의 양이 알고리즘보다 끼치는 영향이 크다. 

충분한 양이 존재하는 데이터여야 함
대표성이 있는 훈련 데이터여야 함
낮은 품질의 데이터는 피해야 함
과적합을 피하는 일반성을 띄되 너무 떨어지지 않게 주의해야 함
적절한 규제가 있는 모델 필요

테스트와 검증을 통해 신뢰성을 확보 해야함
그러나 데이터에 따라 검증 결과 자체도 새 데이터에 따라 점수가 달라질 수 있다. 

홀드아웃 검증
훈련 세트의 일부를 떼어내어 여러 후보 모델을 평가하고 가장 좋은 하나를 선택
반복적으로 교차 검증 수행(비용 높다)


1. 명시적 프로그래밍과 달리 데이터를 통해 규칙을 생성하는 방식의 프로그래밍 
2. 분류 문제(개, 고양이 분류), 예측 문제(시청한 영상에 따라 추천), 인식 문제(복잡한 이미지에 대한 인식), 판단 문제(게임에서 승리하기 위한 에이전트 생성) 
3. 지도 학습을 위해서 사용하는 데이터로 정답이 있는 데이터를 말한다. 
4. 분류와 회귀 
5. 클러스터링이라고 하는 군집화 기법으로 레이블이 없는 데이터에 대해 레이블을 생성하거나 이상치를 확인하고 패턴을 발견하는 등으로 쓰일 수 있다. 
6. 계속해서 실행하는 강화학습이 필요하다. 
7. 군집화가 필요하다. 
8. 어떤 메일이 스팸인지 알 수 없음으로 비지도 학습의 문제의 유형이다. 
9.  온라인 학습이란 실시간 학습으로도 불리며 새로 들어오는 데이터에 대해 즉시 학습하는 방식의 학습 시스템으로 특징으로는 출시 후에도 계속해서 학습을 진행한다는 점과 빠르게 학습을 한다는 점이 있다. 예시로는 매번 그 순간순간 학습을 진행해야하는 네비게이션 최단경로 찾기 같은 부분이 있다. 
10. 외부 메모리 학습은 방대한 데이터를 조금씩 읽어들이는 방식으로 학습하는 시스템
11. 사례기반 측정 
12. 모델 파라미터는 모델이 학습을 통해 자동으로 조정하는 값인데 반해 하이퍼 파라미터는 모델 학습 과정 외부에서 설정되는 값이다. 이런 설정을 우리가 하게 되는 것이다. 
13. 주어진 데이터에 적합한 모델을 학습하는 것으로 데이터로부터 패턴이나 관계를 추정하고 이를 새 데이터에 적용시키는 목적이있다. 성공을 위한 다양한 최적화 방법과 지도 학습, 비지도 학습 같은 방법이 존재하고 예측을 만들기 위해 모델을 학습하며 새 데이터에 대해서 모델이 효용성이 있는지를 평가하는 절차를 반복한다. 
14. 주요 도전 과제는 데이터의 양, 대표성이 있는 데이터를 가지고 훈련해야 한다는 부분과 높은 품질의 데이터를 사용하여 과적합이 심하거나 너무 일반화가 되지 않은 모델을 학습해야한다는 도전과제가 존재하다. 
15. 이는 과적합이 문제일 수 있음으로 이를 막기위해서 데이터 셋을 테스트 셋과 훈련 셋으로 분리 시키는 방법, 모델의 새 데이터에 대한 정확도가 높아지지 않으면 일찍 학습을 종료시키는 방법이 있다.  그 외에도 기존의 훈련 데이터를 인위 적으로 회전시키거나 밝기를 조절하는 등 일반 데이터를 인위적으로 늘려서 일반화 성능을 개선 시키는 데이터 증강과 과도한 가중치를 억제하는 정규화 기법이 존재한다. 
16. 테스트 세트는 기존 데이터에서 학습과 분리시켜 놓은 데이터들의 모음으로 모델이 과적합되었는지 너무 일반화된 모델인지 확인하기 위해서 즉, 좋은 모델인지 판단하기 위해서 사용한다. 
17. 검증 세트는 모델의 성능을 평가하고, 적절한 하이퍼파라미터를 찾기위해서 활용한다. 
18. 훈련-개발 세트(Training-Development Set)는 모델 학습과 하이퍼파라미터 조정, 성능 평가를 위한 중요한 데이터 분할입니다. 훈련 세트는 모델 학습에 사용되고, 개발 세트는 모델 성능을 평가하고 하이퍼파라미터를 최적화하는 데 사용됩니다.
19. 모델이 테스트 세트의 패턴을 이미 알고 있기 때문에, 실제로 모델의 일반화 성능을 정확하게 평가할 수 없습니다
